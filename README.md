# Stock Market Portfolio Management System

## Overview
A relational database system designed to manage stock market portfolios, investor holdings, and transaction records using SQL.

## Objective
To build a structured database that supports portfolio tracking, stock transactions, and performance evaluation.

## Tools & Technologies
- PL/SQL
- Relational Database Design
- ER Modeling
- Data Normalization

## Core Features
- Investor and portfolio management
- Stock buy/sell transaction tracking
- Portfolio valuation logic
- Historical transaction storage
- Data integrity using primary and foreign keys

## Key Learnings
- Designing normalized financial databases
- Writing complex SQL queries
- Structuring financial data efficiently
- Understanding real-world portfolio management systems

## Use Case
This project simulates a simplified stock market portfolio management system similar to those used by brokerage and investment platforms.
